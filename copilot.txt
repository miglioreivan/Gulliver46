### Problema individuato (breve)
Hai già una buona protezione: nello `spawnStationGroup()` cerchi posizioni valide evitando l’edificio, il monitor e una **area di rispetto attorno al bus** (`let busBuffer = 60;`). Tuttavia le dimensioni della fermata (`areaW`, `areaH`, `sidewalkH`) e il **busBuffer** sono valori fissi che diventano troppo grandi su schermi piccoli, così la `while` può fallire o generare posizioni troppo vicine alla fermata precedente o addirittura sovrapposte al bus.  
> “Cerchiamo un'area che possa contenere sia il marciapiede (sopra) che il posteggio (sotto)”.   
> `let busBuffer = 60;` 

---

### Strategia di correzione (sintesi in tabella)

| **Intervento** | **Perché** | **Impatto** | **Difficoltà** |
|---|---:|---:|---:|
| Ridimensionare `areaW/areaH/sidewalkH` in % di `width/height` | Evita fermate troppo grandi su mobile | Alto | Bassa |
| Calcolare `busBuffer` dinamico da `max(bus.w,bus.h)` e `isMobile` | Buffer proporzionale alla dimensione reale del bus | Alto | Bassa |
| Aggiungere **minDistanceFromPrev** per evitare spawn vicini alla fermata precedente | Previene spawn troppo vicino alla tappa precedente | Medio | Bassa |
| Aumentare `safetyCounter` fallback e fallback posizionale (sposta la fermata lontano dal bus) | Evita loop e spawn sovrapposti se non trova posto | Medio | Bassa |
| Controllo post-spawn: se overlap con bus, sposta la fermata in direzione opposta | Ultima linea di difesa runtime | Alto | Media |

### Patch consigliata (sostituisci la funzione `spawnStationGroup()` con questa versione)
```js
function spawnStationGroup() {
    waitingPeds = [];
    bloodSplats = [];

    // Dimensioni adattive: percentuali dello schermo
    let areaW = constrain(floor(width * 0.35), 100, 220); // larghezza fermata adattiva
    let sidewalkH = constrain(floor(height * 0.12), 60, 140); // marciapiede
    let areaH = constrain(floor(height * 0.14), 80, 180); // posteggio
    let totalH = sidewalkH + areaH + 70;

    // Numero pedoni come prima
    let numStudents = 0;
    if (currentStationIndex === 0) {
        numStudents = floor(random(7, 10));
    } else {
        let increase = floor(random(2, 6));
        numStudents = lastPedestrianCount + increase;
    }
    lastPedestrianCount = numStudents;

    // Buffer dinamico attorno al bus (proporzionale alla dimensione reale del bus)
    let busSize = max(bus.w, bus.h);
    let baseBuffer = busSize * 0.9; // ~90% della dimensione maggiore
    let isMobile = width < 500;
    let busBuffer = baseBuffer + (isMobile ? 20 : 0); // mobile: un po' più prudente

    // Evita spawn troppo vicino alla fermata precedente
    let minDistanceFromPrev = isMobile ? width * 0.18 : width * 0.12;
    let prevStationCenter = (typeof stationZone !== 'undefined') ? (stationZone.x + stationZone.w / 2) : -9999;

    let sx = 0, sy = 0;
    let validArea = false;
    let safetyCounter = 0;
    let maxAttempts = 300;

    while (!validArea && safetyCounter < maxAttempts) {
        safetyCounter++;
        sx = random(30, width - areaW - 30);
        sy = random(80, height - totalH - 100);

        // Evita area monitor e edificio come prima
        let monAreaX = width - 200;
        let monAreaY = 320;
        if (sx + areaW > monAreaX && sy < monAreaY) continue;
        if (sx + areaW > univpmBuilding.x - 20 && sx < univpmBuilding.x + univpmBuilding.w + 20 &&
            sy < univpmBuilding.y + univpmBuilding.h + 20) continue;

        // Evita spawn troppo vicino alla fermata precedente (asse X)
        if (prevStationCenter > -9000 && abs((sx + areaW/2) - prevStationCenter) < minDistanceFromPrev) continue;

        // Area rettangolare di rispetto attorno al bus (usiamo busSize per robustezza)
        let busRect = {
            x: bus.x - busSize/2 - busBuffer,
            y: bus.y - busSize/2 - busBuffer,
            w: busSize + busBuffer*2,
            h: busSize + busBuffer*2
        };
        let stationRect = {
            x: sx - 20,
            y: sy - 20,
            w: areaW + 40,
            h: totalH + 40
        };

        // Controllo overlap semplice e robusto
        if (!(stationRect.x + stationRect.w < busRect.x ||
              stationRect.x > busRect.x + busRect.w ||
              stationRect.y + stationRect.h < busRect.y ||
              stationRect.y > busRect.y + busRect.h)) {
            // c'è overlap -> prova a spostare la fermata lateralmente lontano dal bus
            // calcola direzione opposta e sposta di un passo
            let dirX = (stationRect.x + stationRect.w/2) < (busRect.x + busRect.w/2) ? -1 : 1;
            sx += dirX * (busBuffer + areaW * 0.6);
            // se esce dallo schermo, skip e continua
            if (sx < 30 || sx + areaW > width - 30) continue;
            // ricalcola stationRect e ricontrolla
            stationRect.x = sx - 20;
            if (!(stationRect.x + stationRect.w < busRect.x ||
                  stationRect.x > busRect.x + busRect.w ||
                  stationRect.y + stationRect.h < busRect.y ||
                  stationRect.y > busRect.y + busRect.h)) {
                continue; // ancora overlap -> prova un'altra posizione
            }
        }

        // Se siamo arrivati qui, la posizione è valida
        validArea = true;
    }

    // Se non troviamo posizione dopo tanti tentativi, forziamo uno spawn sicuro lontano dal bus
    if (!validArea) {
        // posiziona la fermata sul bordo opposto dello schermo rispetto al bus
        if (bus.x < width / 2) sx = width - areaW - 40;
        else sx = 40;
        sy = constrain(height * 0.2, 80, height - totalH - 100);
    }

    waitingArea = { x: sx, y: sy, w: areaW, h: sidewalkH };
    stationZone = { x: sx, y: sy + sidewalkH, w: areaW, h: areaH };

    // spawn pedoni
    for (let i = 0; i < numStudents; i++) {
        let px = waitingArea.x + random(10, max(10, waitingArea.w - 10));
        let py = waitingArea.y + random(10, max(10, waitingArea.h - 10));
        waitingPeds.push(new Person(px, py));
    }
}
```

---

### Perché queste modifiche risolvono il bug
- **Dimensioni adattive**: `areaW/areaH/sidewalkH` ora dipendono da `width/height`, quindi su telefoni la fermata è più piccola e non “invade” lo spazio del bus.  
- **Buffer proporzionale**: `busBuffer` calcolato da `max(bus.w,bus.h)` evita che un valore fisso (60) sia troppo piccolo o troppo grande rispetto al bus reale.  
- **Minima distanza dalla fermata precedente** evita spawn “attaccati” alla tappa precedente.  
- **Fallback post-spawn** e aumento `maxAttempts` prevengono loop e garantiscono sempre una posizione sicura anche in layout stretti.
